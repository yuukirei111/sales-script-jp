# winning-patterns 写入规范(给 Skill 自己看)

本文件指导 Skill 如何把"用户喂入的爆款脚本"沉淀进 `winning-patterns.md`。

## 触发条件

模式 D 被激活:用户在 B/C 模式后粘贴一段或多段日文脚本文本。

## 解析步骤(每条脚本)

1. **元数据抽取**
   - 来源行号(在 TTS 表里第几行)
   - 账号(梨花 / 玛丽娜)
   - 商品(D 列)
   - 出单数 / CTOR / 播放(J / I / F 列)
   - 视频链接(C 列)
   - 沉淀日期

2. **结构解构**(逐句拆解,标 0–3s / 3–8s / 8–13s / 13–20s)

3. **诉求归类**:对照 `formulas.md` 三大诉求 × 10 子类,判定属于哪一类。**不能归类的标 `[新角度]`,需要追加进新发现区**。

4. **金句去重**:每句话 vs `formulas.md` 全文,**完全匹配** → 已有,跳过;**部分匹配** → 视为变体,加入对应区;**完全新颖** → 加入「新增金句」区。

5. **拟声 / 情绪词盘点**:统计本条出现了哪些 `formulas.md` 列出的词,以及任何**未列出的新词**。

## 写入操作(覆写 `winning-patterns.md`)

### A. 追加单条爆款档案

在「单条爆款档案」section 末尾追加:

```markdown
## #{N} | {YYYY-MM-DD} | {账号} | {出单数}单 | {商品}

**视频链接**:{C列内容}
**指标**:播放 {F} / 完播 {G} / CTOR {I} / 出单 {J}
**归类**:{诉求类型} - {子类}({若属新角度,标 [新角度] 并简述})

**结构拆解**
- 0–3s 钩子:`{原句}`
- 3–8s 展开:`{原句}`
- 8–13s 说服:`{原句}`
- 13–20s CTA:`{原句}`

**关键词审计**
- 用到的 formulas 金句:{列表}
- 新增金句:{列表 → 同步加入下方"新增金句"区}
- 拟声 / 情绪词:{列表}

**用户后续标注**(可空,留给用户回填上线后的复盘)
```

### B. 更新「高权重公式排行榜」

- 每出现一次 `{诉求}.{子类}`,验证次数 +1
- 平均出单数滚动重算
- 表格按累计出单数降序

### C. 增量「新增金句」

- 完全新颖的金句 → 按时段加入对应区
- 在金句末尾标 `(来自 #{N} {商品})` 便于溯源

### D.「新发现的角度」(若有)

- 简要描述新角度逻辑、举例脚本、潜在适用场景
- 标 **观察期**:连续 2 次出现且都 >3 单 → 提议升级为 formulas.md 的新子类(需用户确认)

### E. 更新元数据

- "最近更新" → 写入今天日期
- "累计沉淀爆款数" +1

## 写完后的 Skill 输出

```
✅ winning-patterns.md 已更新(本次沉淀 #{N})
- 新增金句 X 条
- 强化公式 Y 条({诉求.子类} 累计验证 Z 次)
- 新发现角度 W 条({描述})

下一次写脚本会优先调用本周新模式。
```

## 审计 / 回滚

- 每次写入前,read 现有文件 → 找到最大 `#{N}` → N+1
- 写入失败 → 输出错误 + 回滚草稿
- 用户后续若想撤销某条 → 直接编辑 `winning-patterns.md`(它就是个 markdown,可手动改)
