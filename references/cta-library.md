# CTA 库 — Call To Action Library

> 本文件是 TTS 卖货脚本**结尾 CTA(13-20s)** 的唯一弹药池。
>
> 生成脚本时**必读本文件**,候选脚本的 CTA**强制从库存调取** — 不在库的 CTA 需先入库。
>
> 更新机制(同 hooks-library.md):
> 1. **用户口头追加**:用户说「把这个 CTA 加入库存」时,Skill 立即追加,带元数据
> 2. **模式 D 自动入库**:用户喂入爆款时,Skill 自动提取末段 CTA 入库
> 3. **模式 E 评分入库**:评分 ≥85 的脚本 CTA 自动入库
> 4. **永不删除旧条目**

## 使用规则(Skill 生成脚本时必读)

1. **强制从库存调取**:候选脚本的 CTA 必须有匹配的 CTA-ID
2. **多候选要素多样化**:N 条候选的 CTA 必须从不同 CTA-ID 抽取
3. **R11 默认款优先**:无特殊需要时,首选 CTA-001 軟化型
4. **诉求 × 调性双重过滤**:先按"适用诉求"过滤,再按 IP / 调性匹配
5. **配合提示必读**:有些 CTA 需要前段配套(`枚数限定だから』需要前段铺稀缺三层)
6. **反例库必查**:生成后扫一遍 NG-CTA,确保不踩坑
7. **新爆款后强制更新**:模式 D 喂入爆款后,自动提取末段 CTA 入库
