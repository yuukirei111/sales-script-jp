# 钩子库 — Hook Library

> 本文件是 TTS 卖货脚本**开头钩子(0-3s)** 的唯一弹药池。
>
> 生成脚本时**必读本文件**,候选脚本的钩子**强制从库存调取** — 不在库的钩子需先入库。
>
> 更新机制:
> 1. **用户口头追加**:用户说「把这个钩子加入库存」时,Skill 立即追加,带元数据
> 2. **模式 D 自动入库**:用户喂入爆款脚本时,Skill 自动提取首段钩子入库
> 3. **模式 E 评分入库**:评分 ≥85 的脚本钩子自动入库
> 4. **永不删除旧条目**(同 winning-patterns.md 原则)

## 使用规则(Skill 生成脚本时必读)

1. **强制从库存调取**:候选脚本的钩子必须有匹配的 HOOK-ID。不在库的钩子需先入库
2. **多候选要素多样化**:生成 N 条候选时,N 条钩子必须从不同 HOOK-ID 抽取(参见 [[feedback_corpus_variety]])
3. **验证状态优先级**:✅✅✅ > ✅✅ > ✅ > ⭐ > 🔍 > 👀。低数据条目仅在用户明确要求探索时使用
4. **诉求 × IP 双重过滤**:先按"适用诉求"过滤,再按"适用 IP"过滤,剩余候选随机/轮换
5. **配合提示必读**:不只看 hook 本身,要看「配合提示」字段保证骨架完整
6. **新爆款后强制更新**:模式 D 喂入爆款后,自动提取首段钩子并追加(若已有 ID 则更新验证状态)
